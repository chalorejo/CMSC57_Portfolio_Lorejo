Example I Tried in Counting and the Things I've Learned

When working on a password generator, I realized how counting helps measure security. For example, an 8-character password made of lowercase letters has 26826^8 combinations, which is over 200 billion! That’s a good reminder that even small changes in length or character set can make systems much harder to crack.

This topic trained me to think in terms of possibilities and limits. It helped me understand algorithm complexity and made me more thoughtful when designing solutions that involve large datasets or decision trees.

Example I Tried in Algebraic Structures and Groups and the Things I've Learned

A Rubik’s Cube is a real-world group, each twist is like applying a group operation. Solving the cube means understanding how these moves interact. Another cool example: in robotics, transformations form groups that control how robots move and return to a starting position.

This topic helped me see structure in systems. I now recognize how operations relate to each other and how that relationship creates predictability in code and design.

Example I Tried in Groups, Rings, and Fields and the Things I've Learned

I looked into CRC (Cyclic Redundancy Check) for data transmission. It uses arithmetic over GF(2) basically modulo 2, to check for errors. Simple concept, powerful results.

It amazed me how much reliability we get from just a few algebraic rules. Understanding fields and rings showed me that clean, consistent logic is often hiding behind the scenes in systems we take for granted.
